## БЛОК 1: Дашборд родителя (веб)

### Назначение
Единственный интерфейс родителя к данным ребёнка. Открывается в браузере, подключается к часам по BLE, показывает метрики в реальном времени и историю за 24ч. Отдельный Demo-режим для питча инвесторам.

### Стек
- React + Vite + TypeScript
- Recharts (графики)
- Tailwind CSS (стили)
- Web Bluetooth API (связь с часами)
- IndexedDB через `idb` (локальное хранение 24ч у родителя)
- Хостинг: GitHub Pages (`username.github.io/kairo-dashboard/`)

### Палитра
| Элемент | Цвет | Hex |
|---|---|---|
| Фон страницы | тёплый белый | `#FEFCF9` |
| Карточки / панели | молочный | `#F7F3ED` |
| Акцент (графики, кнопки, активные элементы) | терракот | `#D85A30` |
| Текст основной | тёмный индиго | `#1A1538` |
| Текст вторичный | тёплый серый | `#8C8279` |
| Граница карточек | бежевый | `#E8E0D4` |
| Алерт / SOS | красный | `#D84130` |
| Норма / ОК | приглушённый зелёный | `#5A9A6B` |

### Архитектура данных
- Единый интерфейс `DataSource` с двумя имплементациями:
  - `BLEDataSource` — подключение к реальным часам через Web Bluetooth API, BLE-сервис `KairoSpark`, характеристика JSON `{hr, spo2, temp, motion, state, battery, worn}` раз в 5 сек
  - `MockDataSource` — вшитый JSON-сценарий «день из жизни Спарка», тикер 1 точка / 5 сек, можно мотать таймлайн
- Переключатель `[Live BLE | Demo]` в хедере
- При потере BLE — последние данные остаются на экране, статус «Нет связи»
- При восстановлении BLE — ring buffer с часов (24ч) догружается батчем в IndexedDB

### Структура экрана (один экран, скролл вниз)

**1. Хедер**
- Имя ребёнка (захардкожено в MVP: «Миша»)
- Статус устройства: надет / снят / заряжается / нет связи
- Время последнего обновления
- Переключатель `[Live BLE | Demo]`
- Батарея часов (%, иконка)

**2. Spark-аватар (центральный блок)**
- Крупный Spark в контейнере 200×200px (на мобиле 160×160)
- Терракотовый круглый фон (имитация экрана часов), индиго пиксели, молочные блики
- Текущее состояние под аватаром: Calm / Active / Sleepy / Worried
- Строка-вердикт человеческим языком: «Активен, всё в норме» / «Спит, пульс стабильный» / «Повышенный пульс — обратите внимание»
- SVG-код Spark из утверждённого `spark_avatar_4_states_v5` (чат от 2026-05-05)

**3. Четыре KPI-карточки в ряд (на мобиле 2×2 сетка)**

| Карточка | Значение | Формат | Подпись |
|---|---|---|---|
| HR | 82 | `XX уд/мин` + мини-спарклайн за 1ч | «Пульс» |
| SpO2 | 98 | `XX%` | «Кислород» |
| Temp | +0.1 | `±X.X°C` от 14-дневного baseline | «Температура» |
| Шаги | 4 280 | `X XXX` | «Шаги за день» |

- Цвет значения: норма — индиго, отклонение — терракот, алерт — красный
- Педиатрические пороги по возрасту из Fleming et al. Lancet 2011 (HR), Sjoding NEJM 2020 (SpO2), spec 4.3 (temp ±0.2°C от baseline)

**4. График пульса за 24ч**
- Recharts AreaChart, заливка терракот с opacity 0.15, линия терракот
- Ось X: часы (00:00 — 23:59), ось Y: уд/мин (40–160)
- Зоны сна подсвечены серым фоном
- Горизонтальная линия — верхний педиатрический порог (возраст-зависимый)

**5. Таймлайн состояний за день**
- Горизонтальная полоса на всю ширину, высота 32px
- Цветовые сегменты:
  - Calm = приглушённый зелёный `#5A9A6B`
  - Active = терракот `#D85A30`
  - Sleepy = лавандовый `#9B8EC4`
  - Worried = красный `#D84130`
  - Снят = серый `#D4CCC0`
- Время под полосой: метки каждые 3ч
- При наведении/тапе — попап с деталями сегмента

**6. Лента событий**
- Вертикальный список карточек, последнее сверху
- Типы событий:
  - 🔴 SOS (длинное нажатие на часах) — красная карточка, время, длительность
  - 🟠 Потеря контакта BLE — оранжевая карточка, время начала/конца
  - 🟡 Низкий заряд (<15%) — жёлтая карточка
  - 🔵 Parent touch отправлен — голубая карточка, время
  - ⚪ Браслет снят — серая карточка
- Максимум 20 событий на экране, остальные «Показать ещё»

**7. Блок действий**
- Кнопка «Отправить touch» — при нажатии отправляет BLE-команду на часы, триггерит HAP-03 паттерн (2-импульсное мягкое касание)
- Тогл «Тихий режим» — при включении часы подавляют вибрацию (школа/сенсорный режим)

### Что НЕ в MVP (архитектура поддержит позже)
- Фазы сна (deep/light/REM) — требует ML на часах
- Мульти-ребёнок / мульти-родитель
- Геозоны и GPS (отдельный SKU $49)
- Push-уведомления за пределами вкладки
- Бэкенд / облачное хранение
- История за >24ч

### Адаптивность
- Desktop: одна колонка контента, max-width 640px, центр
- Mobile: та же колонка, KPI 2×2, Spark 160×160
- HTTPS обязателен для Web Bluetooth (GitHub Pages даёт из коробки)

## БЛОК 2: Фронт часов (прошивка Waveshare ESP32-S3 AMOLED 2.06")

### Назначение
Единственный интерфейс ребёнка. На экране живёт Spark — пиксельный персонаж, который отражает состояние ребёнка. Ребёнок НЕ видит цифр, графиков, метрик. Он видит друга на руке.

### Железо (Трек 2 donor)
- Waveshare ESP32-S3 Touch AMOLED 2.06"
- Дисплей: AMOLED 410×502px, touch (не используется в MVP)
- SoC: ESP32-S3 dual-core, WiFi + BLE 5
- IMU: QMI8658 (6-axis, есть на борту)
- PMIC: AXP2101 (зарядка через USB-C)
- Батарея: комплектная LiPo (припаяна)
- Допаянные сенсоры: MAX30102 (PPG, I2C, 0x57), MLX90614 (IR temp, I2C, 0x5A), вибромотор coin (GPIO)

### Палитра экрана часов
| Элемент | Цвет | Hex |
|---|---|---|
| Фон AMOLED | терракот | `#D85A30` |
| Пиксели (глаза, рот, индикаторы) | тёмный индиго | `#1A1538` |
| Блики в глазах | молочный | `#FAF6EC` |
| SOS-режим фон | тёмно-красный | `#8B1A1A` |
| Sleep-режим фон | тёмно-синий | `#1A1538` (или приглушённый `#2D2654`) |

### Экраны (переключение короткое нажатие кнопки, цикл)

**Экран 1: Spark Avatar (главный, по умолчанию)**
- Круглая видимая область Ø22мм в корпусе (маскирует прямоугольный AMOLED)
- Spark по центру: пиксельное лицо на терракотовом фоне
- Текущее состояние определяется автоматически по метрикам:
  - **Calm:** HR в норме, motion низкий, worn = true
  - **Active:** шаги ↑, HR ↑, motion от IMU
  - **Sleepy:** ночь (по часам), HR ↓, motion = 0
  - **Worried:** HR выше педиатрического порога, temp отклонение >0.2°C от baseline, или SOS активирован
- Анимация дыхания: масштаб 1.0 → 1.04 → 1.0, частота зависит от состояния:
  - Calm: 0.5 Гц
  - Active: 1 Гц
  - Sleepy: 0.3 Гц
  - Worried: 0.7 Гц с микро-стопом (рваное)
- Экран всегда включён (AMOLED, терракот фон = много пикселей горят, но ESP32 duty cycle оптимизирован)

**Экран 2: Часы**
- Пиксельные цифры крупно: `HH:MM`
- Шрифт: 8-bit стиль, тот же индиго `#1A1538` на терракоте
- День недели мелко под временем
- Без секунд (экономия энергии — обновление раз в минуту)

**Экран 3: Шагомер**
- Пиксельное число шагов крупно
- Маленькая иконка-ботинок пиксельная
- Обновляется автономно в IMU QMI8658 (step counter в железе)

**Экран 4: Сообщение от родителя**
- Если parent touch получен (HAP-03 через BLE) — пиксельное сердечко на экране 1.5 сек при получении
- На этом экране — история: последнее сердечко + время получения
- Если нет сообщений — «—»

**Возврат к Spark Avatar:** автоматически через 10 сек неактивности на экранах 2/3/4

### Кнопка (одна физическая, на боку корпуса)
| Жест | Действие |
|---|---|
| Короткое нажатие | Следующий экран (Spark → Часы → Шаги → Сообщение → Spark) |
| Длинное 3 сек | SOS: вибро-паттерн Морзе SOS (3 коротких × 3 длинных × 3 коротких), BLE пинг `{"event": "sos", "ts": <timestamp>}`, Spark переходит в Worried, фон краснеет |
| Двойное нажатие | Тихий режим (подавление вибрации в школе). Маленькая пиксельная иконка «🔇» в углу экрана когда активен |

### Жест подъёма запястья (wake-on-wrist)
- IMU QMI8658: gyro Y > 150 deg/s → пробуждение дисплея из deep sleep
- Дисплей гаснет обратно через 5 сек неактивности (если ночь / deep sleep режим)
- Днём — дисплей всегда включён, жест только регистрируется для логирования

### BLE-сервис «KairoSpark»
- UUID сервиса: `0000FF10-0000-1000-8000-00805F9B34FB` (или кастомный)
- Характеристика данных (notify, раз в 5 сек):
```json
{
  "hr": 82,
  "spo2": 98,
  "temp": 36.4,
  "motion": 0.3,
  "state": "calm",
  "battery": 74,
  "worn": true,
  "steps": 4280,
  "ts": 1716720000
}
```
- Характеристика событий (notify, по триггеру):
```json
{
  "event": "sos",
  "ts": 1716720000
}
```
Другие события: `wear_on`, `wear_off`, `low_battery`, `parent_touch_received`
- Характеристика команд (write, от дашборда к часам):
```json
{
  "cmd": "parent_touch"
}
```
Другие команды: `silent_mode_on`, `silent_mode_off`

### Ring buffer (локальное хранение на часах)
- При потере BLE — метрики пишутся во встроенную flash ESP32-S3 (SPIFFS или LittleFS)
- Кольцевой буфер на 24 часа (~512 КБ при записи каждые 5 сек)
- При восстановлении BLE — батч-отправка накопленного
- Формат записи: `[ts, hr, spo2, temp, motion, state, steps]` в бинарном виде (экономия места)

### Wear detection (определение ношения)
- Через quality-of-signal от MAX30102 (PPG)
- Если SNR ниже порога 2 минуты подряд → `worn: false`
- При `worn: false`: не записываем данные сна/активности, Spark на экране показывает «—» вместо лица (или засыпает)
- Production migrate: выделенная cap-sense IC per spec 4.5

### Haptic-паттерны вибромотора
| ID | Название | Паттерн | Триггер |
|---|---|---|---|
| HAP-01 | SOS | 3 коротких × 3 длинных × 3 коротких (Морзе SOS) | Длинное нажатие 3 сек |
| HAP-02 | Goal | 5 импульсов нарастающей интенсивности | Достижение цели шагов |
| HAP-03 | Parent Touch | 2 мягких касания | Команда `parent_touch` по BLE |
| HAP-04 | Bedtime | 3 медленных мягких импульса | По расписанию от родителя (не в MVP) |
| HAP-05 | Low Battery | 4 быстрых импульса | Заряд < 15% |

### Управление питанием
| Состояние | MCU | BLE | PPG | IMU | Дисплей | Ток |
|---|---|---|---|---|---|---|
| Active | 240 МГц | Connected 500ms | Непрерывно | 25 Гц | Spark на экране | ~15 мА |
| Idle | 80 МГц | Connected 2s | Burst 1/мин | 12.5 Гц internal | Static | ~8 мА |
| Sleep | RTC | Advertising 5s | Burst каждые 5 мин | 12.5 Гц internal | Off или static frozen | ~3 мА |
| Deep Sleep | RTC | Off | Off | Motion interrupt only | Off | ~0.3 мА |

Заметка: ESP32-S3 потребляет больше nRF5340 из production spec. Цель 7 дней с комплектной батареей (~200 мАч) — агрессивно, реалистичнее 2-3 дня. Для демо достаточно.

### Прошивка — стек
- Arduino IDE или PlatformIO
- ESP32 Arduino Core от Espressif
- Библиотеки: LVGL или прямой SPI/QSPI-доступ к AMOLED драйверу, SparkFun MAX3010x, Adafruit MLX90614, Wire.h для I2C
- BLE: встроенная ESP32 BLE библиотека (NimBLE предпочтительнее — меньше RAM)
- Файловая система: LittleFS для ring buffer
- OTA: не в MVP (прошивка по USB-C)

## Связи между дашбордом и часами

```
[Часы Kairo]                          [Дашборд родителя]
                                       
Spark Avatar ◄──── состояние ────►  Spark Avatar (зеркало)
Метрики     ────── BLE notify ────►  KPI-карточки
                                       График 24ч
                                       Таймлайн
Ring buffer  ────── BLE batch ────►  IndexedDB 24ч
SOS событие  ────── BLE notify ────►  Красный алерт
                                       
            ◄───── BLE write ──────  Кнопка "Отправить touch"
            ◄───── BLE write ──────  Тогл "Тихий режим"
```

Оба интерфейса показывают одного и того же Spark в одном и том же состоянии. Часы — источник истины (state machine на MCU), дашборд — зеркало. Дашборд никогда не меняет состояние Spark, только отображает.
